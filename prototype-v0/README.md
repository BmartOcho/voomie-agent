# prototype-v0: PressFlow AI

PressFlow AI was a May 7 prototype that validated pre-structured job records against a 4-rule prepress checklist using Vertex AI Gemini 2.5 Flash and MongoDB Atlas. The agent ran a single-shot loop: fetch new jobs from `print_shop.active_jobs`, apply rules (CMYK vs RGB, bleed presence, HP Indigo sheet-size limits, stock routing), update each job's status to either `prepress_ready` or `Hold - Customer Service` with a human-readable note. Inputs were already-structured JSON in MongoDB; outputs were status updates plus agent notes. A Streamlit dashboard polled the same collection so a human supervisor could audit decisions in real time.

What it taught us: the validation problem PressFlow solved was downstream of the actual high-value problem — turning messy customer requests into structured specs in the first place. Voomie pivoted upstream. Reusable patterns lifted from this prototype into Voomie: Vertex AI function-calling dispatch loop, MongoDB resilience helpers (retry + ping), Streamlit polling skeleton with status pills and auto-refresh, and the FastMCP/stdio scaffold. The MCP-as-decoration anti-pattern in this prototype — where `mcp_server.py` is defined but the agent bypasses it and calls MongoDB directly through Vertex function declarations — became Voomie's explicit Definition of Done requirement that MCP be actually invoked over stdio, no decoration.
